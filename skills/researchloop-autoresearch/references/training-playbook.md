# Training Playbook

Use this for laddering, sweep management, pruning, seed control, long-run promotion, checkpointing, reproducibility, and failure recovery.

1. Start with the cheapest batch.
2. Prune losers early.
3. Reproduce any apparent win before promoting it.
4. Keep checkpoints and resume paths explicit.
5. Escalate to performance or divergence debugging if a run stops making sense.
