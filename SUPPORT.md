# Support

How to get help, depending on what you need.

## I have a question or want to discuss an idea

Open a [GitHub Discussion](https://github.com/vukrosic/autoresearch-ai/discussions). This is the right place for:

- "How do I use AutoResearch-AI with my repo?"
- "What's the best way to structure my baseline?"
- "Has anyone tried X with this tool?"
- Proposals for new features (before opening an issue)
- Showing off what you built with it

Discussions are public, searchable, and don't put pressure on the maintainer to respond immediately. Other users (and AI agents reading the repo) can learn from the answers.

## I think I found a bug

Open a [Bug report](https://github.com/vukrosic/autoresearch-ai/issues/new?template=bug.yml). Please include:

- the version of `autoresearch-ai` (`autoresearch --version`)
- your Node version (`node --version`) and OS
- the exact command you ran
- what you expected vs. what happened
- output of `autoresearch doctor` if relevant

## I want a new feature

First, check [GOALS.md](GOALS.md) — your idea may already be in the backlog as a `G##` goal. If it is, comment on the matching open issue (or open one using the [Contribute-a-Goal template](.github/ISSUE_TEMPLATE/contribute-goal.yml)).

If it's not in GOALS.md, either:

- Open a [Discussion](https://github.com/vukrosic/autoresearch-ai/discussions) to talk through the idea, or
- Open a [Feature request](https://github.com/vukrosic/autoresearch-ai/issues/new?template=feature.yml) if you've thought it through and know what you want.

## I want to contribute code

Read [CONTRIBUTING.md](CONTRIBUTING.md). The short version: pick a `G##` from [GOALS.md](GOALS.md), claim it via the Contribute-a-Goal issue template, branch, open a PR.

## I found a security vulnerability

**Do not open a public issue.** Email **shifter040@gmail.com**. See [SECURITY.md](SECURITY.md) for the full disclosure process and SLA.

## I represent a research lab or company that wants paid support

Email **shifter040@gmail.com** with your use case. Paid support isn't formally offered yet (the project is alpha), but the maintainer will respond and can usually find a way to help.

## Response time expectations

| Channel | Typical response |
|---|---|
| Security email | ≤ 7 days acknowledgement, ≤ 14 days triage |
| Bug issue with clear repro | ≤ 7 days |
| Feature request | best-effort; may sit in backlog |
| Discussion question | best-effort; community-answered when possible |
| PR review | ≤ 7 days for first review |

The project is maintained part-time. Please be patient and kind.
