# Open Source Plan

ResearchLoop should be open source because trust matters in research workflows.

Users should be able to inspect:

- prompts
- agent rules
- adapters
- repo detection
- run ledger schema
- experiment templates
- report generation

## Repository Goal

The public repo should be simple:

```text
researchloop/
  bin/
  templates/
  docs/
  examples/
  package.json
  README.md
  ROADMAP.md
  CONTRIBUTING.md
  LICENSE
```

## What Stays Open

- npm CLI
- harness generator
- agent prompts
- repo adapters
- local run ledger
- local reports
- examples and templates

## What Could Be Paid Later

- hosted dashboard
- team collaboration
- private run history
- managed GPU queue
- enterprise integrations
- priority support
- lab-specific templates
- private eval and compliance workflows

## Boundary

The open source project should be useful on its own. Paid features should add convenience, scale, and collaboration, not lock away the core research loop.
