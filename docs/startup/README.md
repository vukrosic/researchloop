# Startup Operating System

ResearchLoop is an open source product with a startup motion around it.

Core idea:

Researchers already have repos, models, prompts, evals, and goals. They need a reliable autonomous research loop that agents can follow without losing discipline.

## Who It Serves

- PhD students who need faster ablations and cleaner logs.
- AI labs that need repeatable research loops across repos.
- Companies optimizing prompts, models, evals, agents, or fine-tuning pipelines.
- Independent researchers who want agent help without giving up reproducibility.

## What Startup Means Here

Startup work means building a repeatable engine around the open source tool:

- identify painful research workflows
- talk to real users weekly
- turn repeated pain into product features
- keep the core npm package open source
- charge later for hosting, teams, support, private templates, GPU orchestration, or managed research runs

## Weekly Rhythm

- Monday: choose one product proof and one user-learning goal.
- Tuesday to Thursday: build, test, and talk to users.
- Friday: write a short public update and summarize what users taught us.
- Weekend: optional demo polish, no big strategic pivots unless there is real evidence.

## Key Metrics

- Time from install to first useful prompt.
- Time from install to first logged experiment.
- Number of users who run a second experiment.
- Number of users who share their repo workflow.
- Number of real adapter requests from labs or students.
