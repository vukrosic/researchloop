# ResearchLoop Docs Rules

You are editing `docs/`.

## Scope

This covers:

- `docs/getting-started.md`
- `docs/startup/`
- `docs/competitors/`
- `docs/testing/`
- `docs/research/`
- `docs/site/`

## Rules

1. Keep docs matched to real CLI behavior.
2. Do not describe a workflow that is not actually shipped.
3. If the CLI supports a target or flag, show the exact spelling.
4. If a doc change reflects a behavior change, update the related template or test too.
5. Prefer short, practical examples over broad marketing language.
6. Keep user-facing instructions consistent across README, getting started, and startup docs.

## Review Checklist

- Does the doc match current commands and file paths?
- Does it explain the first useful action clearly?
- Does it separate core product behavior from optional future ideas?
